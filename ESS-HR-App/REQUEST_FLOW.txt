ESS HR APP - REQUEST FLOW
=========================

Data Flow

    Mobile -> Admin:
        - License validation + config (modules, logout)

    Mobile -> Client:
        - Login + business data

    Client -> Admin:
        - Stats (employees, users)


Login Steps

    1. License Check  : POST /ess/admin/api/validate    -> ess_hr_admin
    2. Company List   : GET  /ess/api/auth/companies    -> ess_hr_client
    3. Login          : POST /ess/api/auth/login        -> ess_hr_client
    4. Data Requests  : GET/POST /ess/api/*             -> ess_hr_client


Headers (sent on every request after login)

    Authorization         : Bearer <access_token>
    X-ESS-Company-ID      : <company_id>
    X-ESS-Employee-ID     : <employee_id>
    X-ESS-Login-Mode      : badge | username
    X-ESS-Login-Identifier: <badge_id or username>


Auto Behaviors

    - Token expired (401) : refresh token -> retry request -> else logout
    - Idle too long       : auto logout after autoLogoutDuration hours
